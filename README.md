# 1. Отсутствует возможность регистрации при нажатии на кнопку "Зарегистрироваться"
## Критичность: блокирующая
### Фактический результат:
После нажатия "Зарегистрироваться" в разделе "Мой профиль" возвращается ошибка 404. При этом невозможно начать оформление заказа без зарегистрированного профиля.

### Ожидаемый результат:
После нажатия "Зарегистрироваться" в разделе "Мой профиль" открывается форма регистрации.

### Шаги воспроизведения:
1. Открыть и запустить бот @SelleronOfficialBot.
2. Выбрать в каталоге товар и добавить его в корзину.
3. Открыть корзину и нажать "Перейти к оформлению".
4. После появления предупреждения об отсутствии регистрации закрыть корзину и открыть "Мой профиль".
5. Нажать "Зарегистрироваться".

# 3. Предупреждающее сообщение в корзине, появляющееся из-за невозможности оформить заказ с товарами с разных складов, не исчезает после устранения причины
## Критичность: высокая
### Фактический результат:
Предупреждение о том, что невозможно поместить в один заказ товары с разных складов, сохраняется даже после того, как в корзине остаётся товар только с одного склада. Предупреждение также остаётся при переходе на следующую страницу.

### Ожидаемый результат:
Предупреждение исчезает, как только все товары в корзине принадлежат одному складу.

### Шаги воспроизведения:
1. Открыть и запустить бот @SelleronOfficialBot.
2. Выбрать в каталоге товары с разных складов (например, футболку и кошачий корм).
3. Перейти в корзину и нажать "Перейти к оформлению".
4. Удалить часть товаров, чтобы остались товары только с одного склада, и снова нажать "Перейти к оформлению".

# 4. Ошибка 404 при попытке открыть страницу продавца
## Критичность: критическая
### Фактический результат:
При попытке перейти по ссылке на продавца со страницы товара возвращается ошибка 404.

### Ожидаемый результат:
При переходе по ссылке на продавца на странице товара открывается страница с товарами соответствующего продавца.

### Шаги воспроизведения:
1. Открыть и запустить бот @SelleronOfficialBot.
2. Выбрать в каталоге товар.
3. Нажать на ссылку на продавца внизу карточки товара.
4. Перейти по ссылке, появившейся в переписке с ботом.

# Тестирование API

## 1. Добавленные через запрос API товары не появляются в каталоге
### Критичность: высокая
#### Фактический результат:
При добавлении товаров через запрос API они не появляются в каталоге бота.

#### Ожидаемый результат:
При добавлении товара через запрос API он появляется в каталоге бота и становится доступен для покупки.

#### Шаги воспроизведения:
1. Через Postman сделать запрос POST на адрес https://selleron.ru/api/v1/vendor/create-product/?auth_key=4e9edfc6-8425-453f-9646-c6691f177424 с телом (пример):
   ```json
   { 
        "category_id": 1,
        "subcategory_id": 1,
        "storage_id": 12,
        "departure_method": "by_point",
        "remaining_quantity": 0,
        "title": "Рис",
        "description": "",
        "price": 1.0,
        "size_l": 1.0,
        "size_w": 1.0,
        "size_h": 1.0,
        "size_pack_l": 1.0,
        "size_pack_w": 1.0,
        "size_pack_h": 1.0,
        "weight": 15.0
    }
   ```
2. Запустить бот @SelleronOfficialBot в Telegram и открыть каталог.

## 4. Функция редактирования товара выдаёт ошибку 400 на корректный запрос
### Критичность: критическая
#### Фактический результат:
При отправке полностью корректного запроса возвращается ошибка 400.

*Сценарий 1:* Для исключения возможных ошибок параметры редактируемого товара были полностью скопированы из товара, находящегося в списке товаров на складе (полученном GET-запросом на адрес https://selleron.ru/api/v1/vendor/get_product_list/?auth_key=4e9edfc6-8425-453f-9646-c6691f177424). После этого было внесено изменение в переменную "title". Ответ пришёл с ошибкой 400.

*Сценарий 2:* Ошибка 400 появляется и при отправке запроса без изменений.

#### Ожидаемый результат:
При отправке полностью корректного запроса возвращается ответ со статусом 200.

#### Шаги воспроизведения:
1. Отправить POST-запрос на адрес https://selleron.ru/api/v1/vendor/edit_product/?auth_key=4e9edfc6-8425-453f-9646-c6691f177424, содержащий в теле ID существующего товара.
